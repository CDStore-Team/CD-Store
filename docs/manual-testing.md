## 0. Журнал змін

### Виправлені баги (з попередньої версії)

| Старий ID | Опис | Виправлення |
|-----------|------|-------------|
| BUG-001 | Розрив між Supabase auth та Zustand currentUser | Додано `setCurrentUser` до store + синхронізація в `useAuth` |
| BUG-002 | /sell та /checkout показують порожню сторінку | Тепер коректно перенаправляють на `/auth` (navigate працює) |
| BUG-003 | Сторінка 404 англійською мовою | Тексти змінено на українські у попередньому коміті |
| BUG-006 | Фільтр ціни приймає інвертований діапазон | Додано clamping: min не може бути більше max і навпаки |

### BUG-004: Фільтри відкриті за замовчуванням на mobile/tablet

**Пріоритет:** Medium  
**Статус:** Open  
**Компонент:** `src/pages/CatalogPage.tsx:17`

**Опис:**  
Стан `showFilters` ініціалізується як `true`. На mobile та tablet фільтри займають значну частину екрану, примушуючи користувача прокручувати вниз для перегляду контенту.

**Рекомендація:** `const [showFilters, setShowFilters] = useState(window.innerWidth >= 1024)` або використати хук `useMobile`.

---

### BUG-005: Неіснуючий продавець показує загальний каталог

**Пріоритет:** Medium  
**Статус:** Open  
**Компоненти:** `src/pages/SellerPage.tsx`, `src/pages/CatalogPage.tsx`

**Опис:**  
При переході на `/seller/nonexistent` відображається загальна сторінка каталогу з заголовком "Знайди свій диск" та "Нічого не знайдено / Спробуйте змінити фільтри".

**Рекомендація:** Додати перевірку `seller` у `CatalogPage` і показувати "Продавець не знайдений" або перенаправляти на 404.

### BUG-007: Поля вводу без асоційованих label

**Пріоритет:** Low  
**Статус:** Open  
**Компонент:** `src/pages/CatalogPage.tsx`

**Опис:**  
Поля фільтрів (пошук виконавця, мін. ціна, макс. ціна) мають `<label>` елементи, але без атрибутів `for`/`id` для зв'язку.

**Рекомендація:** Додати `id` до `<input>` та `htmlFor` до `<label>`.

### BUG-008: Відсутній `<main>` landmark елемент

**Пріоритет:** Low  
**Статус:** Open  
**Компонент:** `src/App.tsx`

**Опис:**  
Сторінка не містить `<main>` landmark елемент. Ускладнює навігацію для screen readers.

**Рекомендація:** Обгорнути `<Routes>` у `<main>` в `App.tsx`.
